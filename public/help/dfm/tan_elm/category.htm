Tangency Elimination

Tangency Elimination is a read-write DFM action designed to eliminate slivers between nearly parallel lines with narrow overlaps. If overlap is less than defined in the Min Essential Overlap parameter, lines will be added to widen the overlap.

Tiny pinholes between lines can be eliminated by this action. The Eliminate Pinhole Max Width parameter defines the maximal size of pinholes that can be eliminated.

This action operates on any layer but is designed primarily for signal layers.

 This action works on original features of the layer, not on shapes, such as when cuts and shaves caused by rout and NPTH features are ignored.

Screen Parameters

The parameter Work on: Lines / Surfaces appears in the Tangency Elimination Action screen. Using this parameter you can tell the action what features to work on. Previously, the Tangency Elimination action worked only on non-shaved lines. You can also work also on:

*        Tangent Surfaces, or surfaces that have an overlap less then that defined by the pp_min_overlap parameter.

*        Tangent shaved Lines, or lines that have an overlap less then that defined by the pp_min_overlap parameter.

In order to achieve the defined overlap, the action places fill lines with the .sliver_fill attribute in the necessary locations. Surfaces whose limits dimensions are less than the defined overlap will not be processed.
