ERF Variable

Description

v_consider_board_net

 

This variable allows you to consider a same board net as a same net during a search for spacing violations.

Yes = the action considers a same board net as a same net.

No = the action does not consider board nets.

Example: two pads are connected to one board net, but are not connected on the same layer (which means that they belong to two different layer nets). Even if the spacing between these pads is less then the specified spacing value, this will not be reported as a spacing problem. But, if the ERF variable self_spacing = Yes, this problem is reported in the Same Net Spacing category.

Note: When v_consider_board_net = Yes, the action calculates the step's board nets automatically. This can increase action run time.
