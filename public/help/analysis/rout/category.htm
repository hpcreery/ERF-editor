Rout Layer Checks

The Rout Layer Checks action is a read-only action intended to find potential manufacturability defects in the Rout Layer. This analysis action is designed to check chained rout layers, but can also process design data (i.e. raw routing data as sent from the customer) before any adjustments are made to the data, such as assign chains, etc.

This action requires a special license option: routchk.

 

 

This action can be applied to all layer features or to a selected sequence (a selected sequence contains at least one selected feature). The action gets a rout layer (not necessary in board context).

It will process Lines and Arcs features in these layers. If the selected feature has a .comp attribute, compensation will be taken into

consideration. If the selected feature does not have this attribute, compensation = none is assumed.

 

Chain Checks

 

This action performs chain checks. Listed below are the current types of chain checks:

1. Find locations that will create problems for the tool size (defined or default):

• Arcs that are smaller than the tool size (Small Arc)

• Places where two features are closer than the tool size (Too Narrow).

• Lines whose length is less than the tool size (Short Line).

• Arcs whose length is less than the tool size (Short Arc).

2. Report about features that intersect (go over) each other (Intersection).

 

Note An unfinished chain is not processed and is reported in a special category - Unfinished Chain.

An Unfinished Chain is a chain where only some of the chain’s features are assigned rout chain attributes.
